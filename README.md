# 基于大语言模型的情绪记录与管理智能体系统

## 项目简介

本项目是本科毕业设计项目，题目为《基于大语言模型的情绪记录与管理智能体系统设计与实现》。

系统面向大学生等普通用户，围绕“情绪日记记录、情绪分析、AI 建议生成、情绪趋势可视化”展开。用户可以记录每日情绪日记，系统对文本进行基础情绪分析，并调用大语言模型 API 生成温和、非医疗性质的情绪调节建议。

本系统定位为情绪记录与辅助管理工具，不用于心理疾病诊断或治疗。

## 技术栈规划

| 层级 | 技术 |
| --- | --- |
| 前端 | Vue 3、Element Plus、Axios、ECharts |
| 后端 | Spring Boot、MyBatis-Plus、JWT |
| 数据库 | MySQL |
| AI 能力 | 后端调用大语言模型 API，后续加入简单 NLP 情绪分析 |
| 接口风格 | RESTful API |

## 核心功能规划

### 用户端功能

- 用户注册、登录、退出登录
- 情绪日记新增、查询、修改、删除
- 情绪极性分析：积极、消极、中性
- 情绪类型识别：开心、焦虑、悲伤、愤怒、疲惫、平静等
- 情绪强度评分：1 到 5 分
- 关键词提取
- AI 情绪建议生成
- AI 陪伴对话
- 情绪趋势图、情绪分布图、情绪强度图
- 周报、月报初步生成

### 管理端功能

- 用户管理
- 日记数据统计
- 敏感词管理
- 用户反馈管理

## 智能体设计思路

本项目中的智能体不是复杂的通用 Agent，而是面向情绪记录场景的轻量级智能体。它主要具备以下能力：

| 能力 | 系统体现 |
| --- | --- |
| 感知能力 | 接收用户输入的情绪日记、心情标签和聊天内容 |
| 分析能力 | 对文本进行情绪分类、强度评分和关键词提取 |
| 记忆能力 | 保存用户历史日记、情绪分析结果和对话记录 |
| 反馈能力 | 根据当前情绪和历史记录生成 AI 建议 |
| 可视化能力 | 使用图表展示情绪变化趋势 |

## 最低可完成版本

为了保证毕设能够稳定完成，第一版优先实现以下功能：

- 用户注册和登录
- 情绪日记增删改查
- 基础情绪分析
- AI 建议生成
- 历史记录查看
- 情绪趋势图展示
- 基础后台管理

## 进阶功能

如果基础版本完成较顺利，可以继续扩展：

- AI 陪伴聊天
- 周报和月报生成
- 高频关键词词云
- 情绪日历
- 敏感风险词检测
- 更复杂的 NLP 情绪分析方法

## 开发原则

- 先完成核心闭环，再增加扩展功能。
- 后端统一处理大语言模型 API 调用，前端不直接暴露 API Key。
- 用户日记属于隐私数据，接口需要进行权限控制。
- AI 输出必须避免医疗诊断类表达，只提供日常情绪整理建议。
- 文档、接口、数据库设计保持清晰简单，符合本科毕设实现难度。
