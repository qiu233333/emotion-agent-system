# 开发计划

## 一、项目目标

本项目目标是实现一个基于大语言模型的情绪记录与管理智能体系统。系统以情绪日记为核心，结合基础 NLP 情绪分析、大语言模型建议生成和 ECharts 图表展示，帮助用户记录情绪、理解情绪变化，并获得温和的情绪整理建议。

本项目不做心理疾病诊断，也不替代心理咨询，只作为情绪记录和辅助管理工具。

## 二、开发原则

- 先完成能演示的核心功能，再做进阶功能。
- 先做 Web 端，不优先做小程序和移动端。
- 先使用简单 NLP 情绪分析方法，不一开始使用复杂模型。
- 大语言模型 API 由后端调用，避免前端泄露 API Key。
- 每完成一个模块，及时整理截图、接口说明和测试结果，方便后续写论文。

## 四、核心开发顺序

1. 搭建前端和后端项目。
2. 实现前端请求后端接口。
3. 设计 MySQL 数据表。
4. 实现用户注册和登录。
5. 实现 JWT 登录认证。
6. 实现情绪日记增删改查。
7. 实现基础情绪分析。
8. 接入大语言模型 API 生成建议。
9. 实现 ECharts 情绪统计图表。
10. 完成后台管理和系统测试。

## 五、最低可交付功能

最低版本需要保证可以完整演示以下流程：

1. 用户注册并登录系统。
2. 用户新增一篇情绪日记。
3. 系统保存日记内容。
4. 系统分析情绪极性、情绪类型、情绪强度和关键词。
5. 系统调用大语言模型生成情绪建议。
6. 用户查看历史日记。
7. 用户查看近 7 天情绪趋势图。
8. 管理员查看用户和反馈信息。

该版本已经可以支撑本科毕设论文中的需求分析、系统设计、系统实现和系统测试章节。

## 六、进阶功能安排

基础功能完成后，可以根据时间选择实现以下内容：

| 功能 | 优先级 | 说明 |
| --- | --- | --- |
| AI 陪伴聊天 | 高 | 能体现智能体交互能力 |
| 周报、月报 | 中 | 根据历史日记生成阶段性总结 |
| 关键词词云 | 中 | 增强可视化效果 |
| 情绪日历 | 中 | 展示每日情绪状态 |
| 敏感风险词检测 | 高 | 提升系统安全性 |
| BERT 情感分类 | 低 | 难度较高，时间充足再做 |

## 八、风险与解决方案

| 风险 | 说明 | 解决方案 |
| --- | --- | --- |
| 功能过多做不完 | 毕设时间有限 | 优先完成最低可交付版本 |
| 大模型 API 不稳定 | 接口可能超时或失败 | 增加异常处理，失败时给出默认提示 |
| 情绪分析不准确 | 简单规则可能存在误判 | 在论文中说明局限，后续可扩展模型 |
| 用户隐私数据敏感 | 日记内容属于个人隐私 | 做好用户权限控制和管理端脱敏 |
| 题目涉及心理健康 | 容易被误解为医疗系统 | 明确系统不做诊断和治疗 |
